# Security Policy

## Supported Versions

This project is under active development. Security fixes are provided for the latest version on the `main` branch.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it responsibly.

### Please do

- Open a **private** report by contacting the repository owner/maintainer.
- Provide a clear description, impact, and reproduction steps (or a minimal PoC).
- Include environment details (OS, VS Code/Cursor/Chrome versions, extension version).
- Allow a reasonable amount of time for investigation and a fix before public disclosure.

### Please do not

- Do **not** open a public GitHub issue for vulnerabilities.
- Do **not** post exploit details publicly before coordination.

## Disclosure Process (high level)

1. Maintainer acknowledges receipt of the report.
2. Maintainer investigates and prepares a fix.
3. A patched version is released and users are advised to update.
4. A public note may be published after a fix is available.
